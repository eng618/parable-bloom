# App Store Listings

This directory contains all the documentation, templates, and guidelines needed to prepare and publish **Parable Bloom** to the Apple App Store and Google Play Store.

Keeping these documents up to date ensures a smooth submission process, high-quality store presence, and better App Store Optimization (ASO).

## Directory Overview

* **[metadata.md](metadata.md)**: Guidelines for text-based fields like names, subtitles, descriptions, and keywords.
* **[visual-assets.md](visual-assets.md)**: Specifications and planning for app icons, screenshots, and promo videos.
* **[compliance.md](compliance.md)**: Checklists and details for Privacy Policies, Terms of Service, Data Safety, and age ratings.
* **[marketing.md](marketing.md)**: Press kit contents, release notes templates, and launch preparations.
* **[best-practices.md](best-practices.md)**: ASO recommendations, conversion rate optimization tips, and platform guidelines.
* **[templates/](templates/)**: Blank templates you can duplicate to prepare submissions for new releases.

## Getting Started

When preparing for a new major release or an initial launch:

1. Duplicate the checklists in `templates/ios-checklist.md` and `templates/android-checklist.md`.
2. Fill out a new `templates/app-metadata.md` if any text needs to change.
3. Review the `compliance.md` document to ensure no new data types or permissions have been added to the app that require disclosure.
